**先定目标**

一个月后，你要达到的不是“学完 AI”，而是这三个结果：

1. 能独立做出 RAG 项目
2. 能独立做出一个 Tool Calling Agent 项目
3. 能把项目写进简历，并能应对初中级 AI 应用开发面试

**一个月主学技术栈**

这些不是“了解一下”，而是你这个月最该碰的核心栈。

1. Python AI 应用开发
   - Python
   - requests
   - asyncio
   - pydantic
   - pytest
   - 环境变量、日志、配置管理
2. Web 后端
   - FastAPI
   - REST API
   - 文件上传
   - SSE 流式输出
3. LLM 应用基础
   - Prompt 设计
   - Structured Output
   - Tool Calling
   - 多轮对话管理
   - 上下文管理
4. RAG
   - 文档切分
   - Embedding
   - 向量检索
   - 检索结果拼接 Prompt
   - 引用来源返回
5. Agent
   - LangChain
   - LangGraph
   - 单 Agent 编排
   - 工具注册
   - 失败重试
   - Memory / Recall 基础
6. 工程化与部署
   - PostgreSQL 或 MySQL
   - Redis 基础理解
   - Docker
   - Git
   - Linux 常用命令

**一个月最终项目安排**

这个月只围绕 3 个项目学，别再散学。

1. AI Chat API 小项目
   目标：学会 FastAPI + 模型调用 + 流式输出
2. RAG 知识库问答系统
   目标：学会文档上传、切分、向量化、检索、回答
3. 旅行规划 / 助手型 Agent
   目标：学会 Tool Calling、LangGraph、上下文管理、结果导出

这 3 个项目做完，你就已经有简历雏形了。

**30 天学习计划**

**第 1 周：补齐 AI 应用开发最小基础**
目标：先把“会调模型、会写服务”搞定。

**Day 1**

- 安装并整理开发环境
- 目录统一放到 G:\study\codex-ai
- 建一个总项目目录
- 学 venv、依赖安装、pip freeze
- 复习 Python 文件操作、类、异常处理

**Day 2**

- 学 FastAPI 基础
- 会写 GET、POST
- 会写请求体和返回体
- 理解 pydantic

**Day 3**

- 学大模型 API 调用
- 理解 messages
- 理解 system/user/assistant
- 写一个最小聊天接口

**Day 4**

- 学流式输出
- 用 FastAPI + SSE 做流式返回
- 理解为什么 AI 产品常用流式输出

**Day 5**

- 学结构化输出
- 让模型稳定输出 JSON
- 学会用 pydantic 校验模型输出

**Day 6**

- 做第一个小项目：AI Chat API
- 功能：普通问答、流式问答、JSON 输出

**Day 7**

- 整理第一个项目
- 写 README
- 提交 GitHub
- 复盘这周概念

这周结束你要会：

- 用 Python 写 AI 接口
- 用 FastAPI 提供服务
- 让模型稳定返回结果

------

**第 2 周：RAG 项目周**
目标：做出一个真正能写到简历上的知识库项目。

**Day 8**

- 学 RAG 是什么
- 搞懂流程：加载文档 -> 切分 -> 向量化 -> 检索 -> 生成

**Day 9**

- 学文档读取
- 处理 pdf/md/txt
- 学文本切分策略

**Day 10**

- 学 embedding
- 学向量库
- 先用本地简单方案，比如 FAISS

**Day 11**

- 实现相似度检索
- 用户提问后先召回相关片段
- 再把片段拼进 prompt

**Day 12**

- 给 RAG 加引用来源
- 回答里带“依据哪段文档”
- 这是简历亮点

**Day 13**

- 完成 RAG 知识库问答系统
- 支持上传文档
- 支持提问
- 支持引用返回

**Day 14**

- 优化项目结构
- 把切分、向量化、检索拆模块
- 写 README 和接口说明

这周结束你要会：

- RAG 整体链路
- 文档切分和检索
- 为什么检索比硬塞全文更合理

**你接下来最该记住的学习原则**

1. 不按课程学，按项目学
   看到不会的再补，不用先把所有理论学全
2. 不沉迷框架
   LangChain、LangGraph 要学，但你先要理解本质
3. 不追求大而全
   一个月只做 2 到 3 个项目，做深一点
4. 每天都要有产出
   代码、笔记、README、项目截图，都算产出
5. 面试导向学习
   你每学一个点，都要问自己：
   “这个能不能写进项目亮点里？”
